# Responsive Layout 模組任務

## 目標

管理 iPad 橫向優先的畫面適配，確保角色、障礙、HUD 與操作按鈕清楚可用。

## 最小可執行任務

- [x] 建立 iPad 橫向版面基準。
- [x] 設定遊戲主要畫面比例與安全邊界。
- [x] 設定首頁在橫向畫面中的主要區塊位置。
- [x] 設定接觸士兵階段的左右按鈕位置。
- [x] 設定跑酷階段的蹲下按鈕位置為左下角。
- [x] 設定跑酷階段的跳躍按鈕位置為右下角。
- [x] 確保跳躍與蹲下按鈕距離足夠，避免誤觸。
- [x] 確保 HUD 不遮擋跑酷判斷區。
- [x] 根據畫面尺寸提供 Runner 可用的地面線與障礙生成位置。
- [x] 直向開啟時顯示旋轉提示。
- [x] 確認不實作完整手機直向玩法。

## 完成標準

- [x] iPad 橫向畫面中操作按鈕清楚可按。
- [x] 角色、士兵與分數顯示不互相遮擋。
- [x] 直向開啟時不進入不完整玩法。
